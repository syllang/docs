# 静态检查

Syl 编译器在编译期执行多项静态检查。这些检查确保硬件设计的正确性，减少仿真阶段的调试时间。

---

## 检查分类

静态检查分为以下几类：

**类型检查。** 检查赋值两侧类型和位宽是否一致、函数调用参数类型是否匹配。

**能力检查。** 检查端口方向是否合规：`in` 端口不能赋值，`out` 端口不能读取。

**驱动检查。** 检查多驱动冲突、缺失驱动、重叠驱动。

**完整性检查。** 检查 `match` 是否覆盖所有枚举变体、端口是否全部连接。

**常量检查。** 检查编译期表达式是否有除零、溢出等错误。
