# 路线图

本文说明 Syl 的中长期规划。路线图反映了当前项目团队的重点方向。

---

## 当前阶段

Syl 处于早期开发阶段。编译器核心功能已经可用：前端解析、语义分析、展开和后端生成。编译器可以处理小型到中型的数字设计。

## 近期目标

### 稳定核心语言

**完整类型系统。** 完善泛型参数的支持，增加类型推断。当前的类型系统已经覆盖常见场景，但还有一些边界情况需要处理。

**更好的诊断信息。** 改进编译器错误信息的可读性。错误信息应该指出问题位置、问题类型和修复建议。

**多文件编译。** 完善包管理和导入系统。当前的 `use` 导入已经可用，但文件发现机制还需要改进。

### 标准库扩展

**总线接口。** 增加 AXI、Wishbone 等常见总线接口的定义。

**FIFO 组件。** 增加同步和异步 FIFO 的标准实现。

**时钟域交叉。** 增加跨时钟域同步的基础组件。

### 工具链

**LSP 完善。** 改进语言服务器的悬停提示、跳转定义和自动补全。当前 LSP 支持基本功能，但响应速度和覆盖率还有改进空间。

**波形调试。** 研究是否可以通过编译器导出调试信息，辅助波形调试。

## 中期目标

### 编译器优化

**编译速度。** 针对大型设计优化编译器速度。当前编译器的增量缓存已经减少了重复编译，但第一次编译对于大型设计可能较慢。

**输出质量。** 改善生成的 SystemVerilog 代码风格。当前输出的 SV 代码功能正确，但风格可以直接。

### 验证支持

**结构化诊断。** 将编译器诊断信息导出为标准格式，便于 CI 系统和编辑器集成。

**形式化验证接口。** 研究在 Syl 编译器中生成形式化验证所需的断言。

## 长期目标

### 语言特性

**有符号整数。** 增加 `SInt<W>` 类型，支持有符号整数运算。

**浮点数。** 研究硬件浮点运算的支持方式。

**高级综合优化。** 研究在编译器级别进行面积和时序优化的可能性。

### 生态系统

**包管理器。** 研究 Syl 包的发布和依赖管理方案。

**第三方 IP 集成。** 完善 `extern cell` 和预编译组件的集成流程。

**仿真支持。** 研究将 Syl 编译器直接连接到仿真工具的可能性。

## 优先级原则

项目决策遵循以下优先级：

1. **正确性优先。** 编译器的输出必须功能正确。在优化之前先确保正确。
2. **诊断优先。** 好的错误信息比好的优化更重要。用户需要理解为什么编译失败。
3. **简单优先。** 语言特性应该简单明确。在添加新特性之前，先确认已有特性是否满足需求。
