# 模式和标准库

本文概览 Syl 的常用硬件设计模式。后续每篇文章深入一个模式，展示具体的 Syl 代码和使用场景。这些模式基于标准库提供的基础组件构建。

---

## 设计模式概述

Syl 的硬件设计模式围绕几个核心概念展开：流接口、流水线、参数化和接口约定。这些模式不是语言内置的语法，而是基于 Syl 的类型系统和标准库构建的可复用设计方法。

## 模式列表

**流接口（Stream）。** valid-ready 握手协议。这是 Syl 模块之间数据传递的标准方式。发送方驱动 `payload` 和 `valid`，接收方驱动 `ready`。标准库提供 `Stream` 接口定义。

**流水线（Pipeline）。** 多级流水线数据通路。标准库提供 `Stage`、`stage_link`、`halt_link` 等组件，用于构建带控制的流水线。流水线中的每一级可以独立暂停和刷新。

**参数化设计。** 使用泛型参数创建可复用的模块。Syl 的泛型支持类型参数（如 `W: Nat`）和域参数（如 `D: Domain`），编译器在实例化时展开具体实现。

**接口约定。** bundle 和 interface 的组合使用，定义模块间的通信协议。接口视图确保连接方向的正确性。

## 标准库组件

标准库提供以下基础组件，位于 `examples/std/` 目录下：

**Stream。** `std.stream` 包中的 `Stream<T>` 接口定义 valid-ready 握手协议。接口包含 `payload`、`valid`、`ready` 三个字段，以及 `source` 和 `sink` 两个视图。

**Stage。** `std.stage` 包中的流水线阶段组件，包括：

- `stage_from_stream`：从 Stream 创建流水线输入阶段
- `stage_to_stream`：从流水线输出阶段创建 Stream
- `stage_link`：连接两个流水线阶段
- `halt_link`：可暂停的流水线连接
- `drop_link`：可丢弃数据的流水线连接

**Pipeline。** `std.pipeline` 包中的多级流水线组合工具。

**Bundle 工具。** `std.bundle` 包中的 bundle 操作工具函数。

**Logic。** `std.logic` 包中的通用逻辑函数，包括位操作和算术辅助。

**Assert。** `std.assert` 包中的编译期断言工具。

## 设计原则

### 模块接口标准化

使用 Stream 接口作为模块间的数据通道。统一的 valid-ready 握手让不同模块可以互相连接，不需要适配逻辑。

### 参数化优先

把可以配置的参数（位宽、深度、使能选项）设计为泛型参数。参数化的模块可以被不同的使用场景复用。

### 组合优于修改

通过组合标准库组件构建复杂设计；标准库组件设计为可组合的构建块。

## 标准库的定位

标准库不是编译器的特殊组件。它是一组用 Syl 编写的普通代码，和其他 Syl 代码一样经过完整的编译流程。这意味着：

1. 标准库没有特权功能。编译器不会对标准库的代码做特殊优化。
2. 标准库可以被用户修改和扩展。如果标准库中缺少某个组件，用户可以自行编写。
3. 标准库的源码是学习 Syl 的参考材料。阅读 `examples/std/` 下的代码可以了解 Syl 的编码风格。

## 模式的选择

在开始设计之前，先确定适用的设计模式：

- 如果模块之间需要传递数据，使用 Stream 接口
- 如果数据通路需要多级处理，使用流水线模式
- 如果模块需要适应不同配置，使用参数化设计
- 如果协议复杂，使用 interface 和 view 封装

## 学习路径

建议按以下顺序学习模式：先掌握流接口（Stream），这是模块间通信的基础。然后学习流水线（Pipeline），理解多级数据通路的构建。之后学习参数化设计，了解如何创建可复用的模块。最后学习接口约定，掌握协议设计的方法。
