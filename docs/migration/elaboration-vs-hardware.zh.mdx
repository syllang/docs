# 展开与硬件

Syl 中有两种不同的计算：展开期计算和硬件计算。理解它们的区别有助于理解 Syl 的编译模型。

---

## 两种计算

Syl 程序中的代码在两种不同的阶段执行。

**展开期（elaboration time）。** 代码在编译时执行。展开期代码不生成硬件。展开期代码可以操作编译期类型（`Nat`、`Bool`）。

**硬件期（hardware time）。** 代码生成实际的硬件电路。硬件期代码在运行时执行。硬件期代码可以操作硬件类型（`Bit`、`UInt<W>`）。

## 展开期构造

### const

```syl
const W: Nat = 8
const ADDR_BITS: Nat = clog2(16)
```

`const` 的值在展开期确定。结果用于类型参数和硬件生成。

### fn

```syl
fn clog2(x: Nat) -> Nat {
    var n: Nat = 0
    var p: Nat = 1
    while p < x {
        p = p << 1
        n = n + 1
    }
    return n
}
```

`fn` 在展开期执行。它接受 `Nat` 或 `Bool` 参数，返回 `Nat` 或 `Bool` 结果。

### 展开期 if

```syl
if W > 8 {
    signal wide: UInt<16>
}
```

展开期 `if` 的条件必须是 `Bool`（编译期布尔值）。条件为 `false` 的分支中的硬件代码不会生成。

### 展开期 for

```syl
for i in 0..4 {
    signal s: UInt<8>
}
```

展开期 `for` 的循环次数必须是编译期常量。循环体中的硬件代码被复制多次。

## 硬件期构造

### signal 和 reg

```syl
signal s: UInt<8> := a + b
reg r: UInt<8> reset(rst, 0)
```

`signal` 和 `reg` 声明在硬件期存在。`signal` 的赋值持续生效。`reg` 在时钟边沿更新。

### := 和 next

```syl
y := x and enable   // 硬件期：连续赋值
next r := r + 1     // 硬件期：寄存器赋值
```

`:=` 和 `next` 都生成硬件电路。`:=` 对应组合逻辑，`next` 对应时序逻辑。

## map

`map` 是一种边界情况。它定义纯组合逻辑，在硬件期执行，但它的调用在展开期被展开。

```syl
map add(a: UInt<8>, b: UInt<8>) -> UInt<8> = a + b
```

`add` 的调用在展开期被替换为组合逻辑表达式。`map` 本身不产生硬件开销（调用点展开为内联逻辑）。

## 展开期和硬件的交互

### 展开期影响硬件生成

展开期的计算结果决定了生成哪些硬件：

```syl
const HAS_DEBUG: Bool = true

cell Top(data: in UInt<8>, debug_data: out UInt<8>) {
    if HAS_DEBUG {
        debug_data := data
    } else {
        debug_data := 0
    }
}
```

`HAS_DEBUG` 是展开期常量。如果它为 `false`，`debug_data := data` 这行代码不生成硬件，减少面积。

### 硬件影响展开期（不直接）

硬件期的值和结构不能反过来影响展开期的计算。例如，不能在展开期循环中依赖硬件信号的值：

```syl
# fn compute(x: Bit) -> Nat { ... }  // 错误：fn 不能接受硬件类型参数
```

## 为什么会混淆

展开期和硬件期的混淆在硬件设计者中很常见。原因是在大多数硬件描述语言中，这两种计算混合在一起。

Verilog 的 `generate` 构造就是展开期计算。但它和普通代码混在同一文件中，新手很难区分哪些代码在展开期执行、哪些生成硬件。

Syl 通过类型系统区分两者：展开期代码操作 `Nat` 和 `Bool`，硬件期代码操作 `Bit` 和 `UInt<W>`。编译器根据操作数类型判断计算阶段。这种设计让阶段区分在代码中可见，不需要依赖注释或约定。对于新手来说，记住这个规则可以避免大多数阶段混用错误。编译器根据操作数类型判断计算阶段。

## 常见混淆

### fn 中操作硬件类型

```syl
# fn add(x: UInt<8>) -> UInt<8> { x + 1 }  // 错误
```

`fn` 只能操作编译期类型。需要操作硬件的逻辑使用 `map`。

### 硬件信号作为展开期条件

```syl
# if some_hardware_signal { }  // 错误
```

展开期 `if` 的条件必须是编译期 `Bool`。硬件信号的值在运行时才知道。

### 展开期计算影响硬件

```syl
const N: Nat = 4
signal arr: [N] UInt<8>  // N 在展开期确定，生成 4 个元素
```

展开期计算正确则硬件生成正确。如果 `N` 在展开期计算错误（例如除零），硬件无法生成。
