# 用户指南

本文说明 Syl 的目标用户和适用场景，帮助你判断 Syl 是否适合你和你的项目。

---

## Syl 适合谁

**学习硬件描述语言的学生。** Syl 的编译器在编译期做完整检查，错误信息明确，适合作为学习数字电路设计的语言。

**探索语言设计的工程师。** Syl 的设计目标之一是静态可分析性。如果你对这个方向感兴趣，Syl 是一个可运行的研究平台。

**快速原型验证的团队。** 如果团队在项目早期需要快速验证硬件架构，Syl 可以在编译阶段捕获连接错误，减少仿真等待时间。

## Syl 不适合谁

**生产项目团队。** Syl 处于早期阶段，编译器可能包含未发现的 bug。不建议用于投片项目。

**需要成熟工具链的团队。** Syl 的 LSP 支持、波形调试、覆盖率分析等工具尚在开发中。如果需要成熟的工具链，建议使用 Verilog 或 VHDL。

**混合信号设计者。** Syl 只支持数字电路描述，不支持模拟电路或混合信号。

## 使用 Syl 的项目类型

Syl 适合以下类型的项目：

- 小型数字模块（计数器、加法器、移位寄存器）
- 有限状态机
- 流接口处理逻辑
- 流水线数据通路
- 协议解析器

## 工作流程

使用 Syl 的典型工作流程：

1. 用 Syl 描述硬件设计
2. 运行 `sylc` 编译器，检查是否有编译错误
3. 根据编译器输出修正类型错误、连接错误和驱动冲突
4. 编译器输出 SystemVerilog 代码
5. 使用标准仿真工具对 SV 代码进行仿真验证
6. 使用综合工具将 SV 代码转换为网表

## 学习路径

如果你是 Syl 新手，推荐以下学习顺序：

1. 阅读"什么是 Syl"了解基本概念
2. 通过"语言基础"章节了解类型系统和语法
3. 查看 `examples/mvp/` 目录中的示例代码
4. 尝试修改示例并观察编译器的输出和错误信息
5. 阅读"模式"章节了解接口、流水线等常用设计模式

## 从其他语言迁移

如果你从其他硬件描述语言迁移到 Syl，以下是对照：

**从 Verilog 迁移。** Syl 的 `cell` 对应 Verilog 的 `module`，`:=` 对应 `assign`，`next` 对应时序逻辑。Syl 的端口方向是强制的，不能省略。`inout` 端口需要特别声明，不会自动推断。

**从 Chisel 迁移。** Chisel 的 `Wire` 对应 Syl 的 `signal`，`RegInit` 对应 `reg reset(rst, init)`，`:=` 运算符的语义类似。Syl 不需要 `import` Chisel 库。

**从 SpinalHDL 迁移。** SpinalHDL 的 `Bool` 对应 Syl 的 `Bit`，`Bits(width)` 对应 `UInt<W>`。SpinalHDL 的 `Area` 对应 Syl 的 `cell`。

## 编写可维护的 Syl 代码

### 命名规范

- 模块名使用大驼峰：`Counter`、`StreamBuffer`
- 端口名和信号名使用小驼峰或下划线：`data_in`、`valid`、`ready`
- 常量名使用大写：`DATA_WIDTH`、`FIFO_DEPTH`
- 类型参数使用大写单字母：`W`、`T`、`D`

### 模块组织

一个文件通常包含一个主要模块和相关类型的定义。紧密相关的模块可以放在同一文件中。跨文件引用使用 `use` 路径导入。

### 使用注释

Syl 代码使用 `#` 行注释。注释应该说明"为什么这样做"——代码本身已经表达了"做了什么"。

## 示例项目

仓库的 `examples/` 目录包含以下参考设计：

- `mvp/counter.syl`：带使能和复位的计数器
- `mvp/lane_array.syl`：通道阵列，展示泛型参数和数组
- `mvp/stream_buffer.syl`：流缓冲，展示接口和视图
- `mvp/com_alu.syl`：组合 ALU，展示 map 和 match
- `pipeline_user.syl`：流水线 ALU，展示标准库的 Stream 和 Stage 使用

## 文件组织建议

Syl 项目的典型文件结构：

```
project/
├── src/
│   ├── top.syl       # 顶层模块
│   ├── counter.syl   # 子模块
│   └── types.syl     # 类型定义
├── vendor/
│   └── ip.syl        # 外部模块声明
└── build.sv          # 编译器输出
```

编译时指定所有源文件：

```
sylc src/top.syl src/counter.syl src/types.syl vendor/ip.syl -o build.sv
```

## 常见错误及处理

### 端口方向错误

```syl
cell Bad(x: in Bit) {
    x := 1  // 错误：in 端口不能被赋值
}
```

`in` 端口由外部驱动，模块内部只能读取。如果需要从模块内部驱动，使用 `out` 方向。

### 类型宽度不匹配

```syl
cell Bad(x: in UInt<8>, y: out UInt<16>) {
    y := x  // 错误：UInt<8> 不能赋值给 UInt<16>
}
```

8 位信号不能直接赋值给 16 位端口。需要使用类型转换或显式扩展。

### 未连接端口

```syl
cell Sub(a: in Bit, b: out Bit) { }

cell Top(y: out Bit) {
    let inst = place Sub(a: y)  // 错误：b 未连接
}
```

实例化时所有端口都需要连接。未连接的端口会被编译器报告为错误。

每个示例都可以单独编译：`sylc examples/mvp/counter.syl`。
