# 项目状态

本文说明 Syl 编译器当前的能力边界。如果你在考虑使用 Syl，这篇文章帮助你判断当前的成熟度是否匹配你的需求。

---

## 验证过的功能

以下功能经过架构合约测试验证，每次 CI 都会运行。

**完整的编译流程。** 从前端到后端到 Verilator 烟雾测试。编译器可以处理包含模块定义、接口声明、组合逻辑和时序逻辑的设计文件，生成 Verilator 可解析的 SystemVerilog 代码。

**参数化模块的实例化和展开。** 模块可以携带类型参数和值参数，实例化时绑定具体值。编译器将参数化模块展开为具体实例，生成各实例对应的硬件。

**名字解析和类型检查。** 编译器正确解析跨文件的引入（`use`），检查表达式类型的一致性，检查端口方向的合规性。

**常量求值。** 编译期执行 `fn` 函数，支持条件分支和循环。求值结果用于计算泛型参数和数组大小。

**驱动分析。** 编译器检测多驱动冲突和缺失驱动。

**不透明摘要。** 外部模块可以通过 `extern cell` 声明签名。预编译的第三方组件可以通过注入摘要来描述内部行为。

**分层诊断。** 编译器按阶段（解析、语义、展开）分组诊断信息。编辑器可以通过 LSP 获取结构化的错误信息。

**缓存。** 编译器缓存每个包的语义分析结果，增量更新时只重新编译受影响的包。

## 验证过但限制较多的功能

**数组类型 `[N] T`。** 支持索引访问和 `for` 循环展开。数组大小必须是编译期确定的常量。

**枚举类型。** 支持 `enum` 声明和 `match` 表达式。枚举变体没有关联数据。

**组合逻辑映射。** `map` 定义纯组合函数。`map` 内部可以使用 `select`、`match` 和算术运算。

**寄存器复位。** `reg` 支持同步复位，在 `reset(rst, value)` 中指定复位信号和复位值。

## 当前能做

**描述组合逻辑。** 使用 `map` 声明纯组合函数，使用 `select` 和 `match` 描述条件选择。

**描述时序逻辑。** 使用 `reg` 声明寄存器，使用 `next` 定义下一状态。

**模块实例化和连接。** 使用 `place` 实例化其他模块，使用命名参数连接端口。

**泛型参数。** 模块和函数支持类型参数（如 `UInt<W>`）和值参数（如 `W: Nat`）。

**接口和视图。** 使用 `interface` 定义端口分组，使用 `view` 定义不同连接方向看到的端口集合。

**流接口。** 标准库提供 `Stream` 接口，支持 valid-ready 握手。

**流水线阶段。** 标准库提供 `Stage` 构造，支持多级流水线。

**SystemVerilog 输出。** 编译器可以生成可综合的 SystemVerilog 代码。在 CI 中通过 Verilator 烟雾测试。

## 实验中的功能

**枚举类型。** `enum` 声明可以定义命名的状态值集合，用于状态机和指令解码。

**数组类型。** `[N] T` 声明一个包含 N 个元素的数组。支持索引访问和 `for` 循环展开。

**编译期计算。** `fn` 函数在编译期执行，用于计算常量值。支持条件分支和循环。

## 还不支持

**黑盒模块。** `extern cell` 可以在 Syl 中声明签名的外部模块，但从 IP 供应商导入黑盒的工作流还在设计中。

**多种时钟域。** 语言支持 `Domain` 类型参数，但多时钟域设计的综合检查尚未实现。

**形式化验证接口。** Syl 不生成断言（assertion）或覆盖（cover）语句。

**高层次综合（HLS）。** Syl 是硬件描述语言，不是 C 到 RTL 的综合工具。

**仿真支持。** Syl 编译器本身不运行仿真。生成的 SystemVerilog 代码使用标准仿真工具。

## 标准库现状

标准库目前包含以下包：

| 包 | 内容 |
|----|------|
| `std.stream` | Stream 接口定义，valid-ready 握手协议 |
| `std.stage` | 流水线阶段：`stage_from_stream`、`stage_to_stream`、`stage_link`、`halt_link`、`drop_link` |
| `std.pipeline` | 多级流水线组合 |
| `std.bundle` | bundle 工具函数 |
| `std.logic` | 通用逻辑函数 |
| `std.assert` | 断言工具 |
| `std.vendor` | 供应商模块签名示例 |

这些包使用 Syl 编写，和其他 Syl 代码一样经过完整编译。

## 工具链现状

**sylc（命令行编译器）。** 接受 `.syl` 源文件，输出 SystemVerilog。命令行的基本功能已经可用。

**LSP 语言服务器。** 支持基本的悬停提示、跳转定义和自动补全。用于编辑器集成。

**模糊测试。** 解析器经过模糊测试，可以处理随机生成的输入而不崩溃。

## 路线图

项目优先做三件事：

1. 稳定核心语言：完成类型系统、完善诊断信息
2. 标准库扩展：增加常用总线接口和组件
3. 工具链集成：完善 LSP、改善编辑器体验

详细的路线图在项目仓库的 roadmap 文档中。
